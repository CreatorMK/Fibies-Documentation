01 — Overview

FIBIES is a collection of 1,123 generative 3D artifacts inscribed on Bitcoin using the parent/child inscription model. Every piece is deterministically generated from its block hash — no servers, no APIs, pure math on pure chain.

The rendering engine is a single inscription — 143KB of vanilla Three.js. Each child inscription references the parent engine and passes its block hash as the generation seed. The result: fully on-chain, fully deterministic, renders forever in any browser.

02 — How It Works

Block Hash → Seed

Each FIBIES piece is seeded by the hash of the Bitcoin block it was inscribed in. This 64-character hex string is the sole input to the generation algorithm. Same hash, same piece — every time, everywhere.

block_hash → seed → type + traits → 3D render

Parent/Child Model

The parent inscription holds the full rendering engine. Child inscriptions are lightweight — they carry only their block hash and a reference to the parent. When loaded, the child fetches the engine from the parent and renders itself.

Deterministic Generation

The seed is hashed through a fibonacci-based PRNG to produce all parameters: type selection, geometry dimensions, color palette, expression, skin pattern, and special traits. No randomness at render time — the output is fixed at inscription.

03 — The Types

Every FIBIES piece belongs to one of █ types, each with distinct geometry, behavior, and rarity tier. Types are assigned by the generation algorithm based on probability weights derived from the block hash.

Rarity Tiers

  • Common — Base types with standard trait pools
  • Uncommon — Modified geometry and expanded trait ranges
  • Epic — Rare types with unique visual signatures
  • Legendary — Extremely rare, one-of-a-kind visual artifacts

Each type has its own set of possible traits: expressions, skins, fibonacci-scaled dimensions, and special visual effects. Trait probability is weighted — some combinations are significantly rarer than others.

05 — On-Chain Engine

The FIBIES engine is a fully self-contained Three.js application inscribed on Bitcoin. It requires no external dependencies, CDNs, or servers.

Technical Specs

  • Size: 143KB (parent inscription)
  • Renderer: Three.js WebGL
  • Dependencies: None — vanilla JavaScript
  • Browser support: Any modern browser with WebGL
  • Generation time: Instant — deterministic, no network calls

The engine handles: seed parsing, type determination, trait generation, 3D geometry construction, material creation, lighting, camera setup, animation loops, and responsive rendering — all from a single file.

06 — Inscription Details

Structure

Parent inscription (engine) ├── Child #1 (block hash → FIBIES piece) ├── Child #2 (block hash → FIBIES piece) ├── ... └── Child #1123 (block hash → FIBIES piece)

Content Type

All inscriptions use text/html content type, allowing them to render directly in any ordinals explorer or compatible viewer. The parent contains the full engine; children contain minimal HTML that references the parent.
